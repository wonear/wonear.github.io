scrapy 安装

安装指令 pip install scrapy

如果在window系统上，还需要额外安装pypiwin32

安装指令 pip install pypiwin32

创建scrapy 项目

在安装了scrapy的环境中执行指令

  • scrapy startproject [项目名称]

scrapy 项目中的目录及其作用

  • items：用来存储爬虫的数据模型
  • middlewares：存放各种中间件的文件
  • pipelines：用来将数据存储到本地磁盘中
  • settings:项目爬虫的各种配置信息
  • scrapy.cfg：项目配置文件
  • spiders文件：爬虫的具体实现，放在这个包中

如果要创建一个爬虫文件，那么先进入spiders文件目录下，在创建，
创建爬虫文件指令 ：

  • scrapy genspider 【文件名】 【要爬取的网站域名】

开始爬虫

1. 先设置配置类settings .py

  1. 一般把配置中的 ROBOTSTXT_OBEY 的值改为False，如果这边的值是True,那么爬虫开始时，首先去爬取网站根目录下的robots.txt文件，如果没有，那么程序就不会去执行接下来的爬虫操作。
  2. 设置请求头

设置好这两步之后就能拿到网站的内容的了，接下来进行数据的解析，拿到自己想要的是数据。

  • 执行指令 scrapy crawl 爬虫文件设置的name
    指令也可以用开发工具带代理执行，新建一个py文件：
from scrapy import cmdline\
cmdline.execute("scrapy crawl qsbk".split(" "))

执行文件也可运行爬虫程序

2. 抓到的对象传到pipelines .py

  1. 在item中定义好数据模型
    import scrapy

class ScraptTestItem(scrapy.Item):
author = scrapy.Field()
content = scrapy.Field()


2. 首先在setting中配置

> ITEM_PIPELINES = {
   'scrapt_test.pipelines.ScraptTestPipeline': 300,
}
 2. 在爬虫代码中这行 **yield** 语句

然后在pipeline文件中写入：

**写法1**
```python
import json


class ScraptTestPipeline(object):

    def __init__(self):
        self.file = open("duanzi.json", 'w', encoding='utf-8')

    def open_spider(self, spider):
        # 开始执行
        pass

    def process_item(self, item, spider):
        # 执行中
        item_json = json.dumps(dict(item), ensure_ascii=False)
        self.file.write(item_json + ",\n\n")

    def close_spider(self, spider):
        # 执行结束
        self.file.close()

注意： 在使用python内置的json模块，对象要转成json需要先把对象转成字典,此时写入的文件的格式何以用 ‘w’

写法2

from scrapy.exporters import JsonItemExporter


class ScraptTestPipeline(object):

    def __init__(self):
        self.file = open("duanzi.json", 'wb')
        self.exporter = JsonItemExporter(self.file, ensure_ascii=False, encoding='utf-8')
        self.exporter.start_exporting()

    def open_spider(self, spider):
        # 开始执行
        pass

    def process_item(self, item, spider):
        # 执行中
        self.exporter.export_item(item)

    def close_spider(self, spider):
        # 执行结束
        self.exporter.finish_exporting()
        self.file.close()

这种方式的缺点是，导出器会把json先存在内存中，在执行self.exporter.finish_exporting时，再统一写入到文件

写法3

from scrapy.exporters import JsonLinesItemExporter


class ScraptTestPipeline(object):

    def __init__(self):
        self.file = open("duanzi.json", 'wb')
        self.exporter = JsonLinesItemExporter(self.file, ensure_ascii=False, encoding='utf-8')


    def open_spider(self, spider):
        # 开始执行
        pass

    def process_item(self, item, spider):
        # 执行中
        self.exporter.export_item(item)

    def close_spider(self, spider):
        # 执行结束
        self.file.close()

这种方式是单个单个的把json写入到文件中

批量爬取

批量爬取只要找到网页中下一页的位置，拿到对应的下一页的连接，在进行拼装，然后重复执行之前的代码就行了

from scrapt_test.items import ScraptTestItem

web_scheme = 'https://www.qiushibaike.com'
class QsbkSpider(scrapy.Spider):

    name = 'qsbk'
    allowed_domains = ['qiushibaike.com']
    start_urls = [web_scheme + '/text/page/1/']

    def parse(self, response):

        # 获取段子div
        divs = response.xpath('//div[@id="content-left"]/div')
        for div in divs:
            contents = div.xpath('.//div[@class="content"]//text()').getall()
            author = div.xpath('.//h2/text()').get().strip()
            content = ""
            for c in contents:
                content = content + c.replace('\n', '')
            item = ScraptTestItem(author=author, content=content)
            yield item

        next_url = response.xpath('//ul[@class="pagination"]/li[last()]/a/@href').get()
        if not next_url:
            return
        else:
            next_url=web_scheme+next_url
            yield scrapy.Request(next_url, callback=self.parse)
